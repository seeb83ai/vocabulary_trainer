# Extract `DetectConfusion` as a testable store method

Status: ready-for-agent
Category: enhancement

## What to build

Rename and expose the confusion pair detection logic as a clearly-named, independently-testable store method. Currently, the 39-line `LookupConfusion` SQL in `service/db/quiz.go` is only reachable by running through the full `QuizHandler.Answer()` HTTP flow. There is no way to ask "is answer X a confusion pair for vocabulary entry Y?" without standing up an HTTP server.

Extract the logic so it can be called and tested directly:

```go
// DetectConfusion checks whether userAnswer is a valid native-language translation
// of a different vocabulary entry than wordID, indicating a confusion pair.
func (s *Store) DetectConfusion(ctx context.Context, userID int64, wordID int64, userAnswer string, mode string, langs []string) (confusedWithID int64, found bool, err error)
```

`QuizHandler.Answer()` calls `DetectConfusion` (replacing the existing `LookupConfusion` call). Rename `LookupConfusion` to `DetectConfusion` — no new abstraction layer needed.
